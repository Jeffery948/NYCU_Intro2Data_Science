Missing Data Overview






  • 由上圖可以發現這份資料的missing values 是用 “” (empty string)表示, 因此我在讀檔案時多加了 na.string = , 來把 “” 替換成 NA



  • 我使用 sum 和 complete 函式去算 8807 個 case 中有多少 case 是有缺失值的, 發現有 3475 個 case



  • 接著我分析了哪些 column 有缺失值並有多少缺失值, 可以看到有缺失資料的分別是 director, cast, country, date_added, rating, and duration, 我將對這些資料進行 imputation

Handle Missing Data by Different Types of Data Imputation Methods

Replace Data with “Unknown”



  • 由於 director 缺失值過多, 且其本身資料性質也不適合進行預測或中位數補值, 因此我選擇將缺失值都改成 Unknown, imputation 後用 sum 和 is_na 確認 director 的缺失值都已經被補上了

Replace categorical data with bootstrap



  • 我使用了 bootstrap 去進行 cast 和 country 的補值處理, 因為它們缺值也較多, 不適合用中位數或平均來補值, 因此我選擇抽樣的方式來補值, 一樣檢查 imputation 是否成功

Most Frequent (mode) Imputation









  • 因為 date_added, rating, duration 的缺失值很少, 所以我把它們出現最多次的取出來補值, 方法是用 sort 配上 table 函式再用 names, 一樣檢查 imputation 是否成功

Summary Data






  • 經過整理之後, 可以發現此資料的缺失值已全部補上, 代表 imputation 成功

Conclusion

  • 我選的資料有缺失值的全部都是 character 型態, 因此我沒辦法做很想做的 mean / median imputation, 雖然可以先將資料轉成 numeric 再進行處理, 但我認為十分麻煩所以還是採取了 most frequent 的方法

  • 我的資料幾乎很難可以用預測或是計算的方式來補值, 像是導演, 演員, 出產國家等, 能正確補值的機率不大, 因此我認為這份資料我所進行的補值不會很準確, 對資料的正確性影響可能頗大

  • 至於 data_added, rating, duration, 由於它們的缺失很少, 因此我認為就算我運氣很差補值剛好全錯, 對整體資料的正確性影響不會很大

Possible Problems

  • 我有嘗試使用 KNN 演算法來進行 data imputation, 但是我始終做不出來, 一下因為維度不同而出錯, 一下發現沒有足夠的 complete case 來計算, 最終我選擇放棄, 我應該會找其他資料來繼續練習 KNN 直到成功為止

  • 前面有提到可以將資料轉成 numeric 來處理, 但因其缺失值太少, 我認為這樣做的效益不會高到哪去, 因此沒有練習到, 若是有機會的話, 我會在之後的作業練習

  • 我很好導演和演員到底能不能依靠預測的方式成功完成高精確度的補值, 我目前的想法是利用機器學習來進行, 但是我不清楚要餵電腦哪方面的資料(ex. 電影類型, 但我選的資料沒有)才能讓它精準預測成功, 而如果導演或演員成功精準的補值了, 那我也許可以更進一步預測 rating, 這是我未來想解決的一大問題
